# Скилл: Первичный приём клиента

Перед выполнением — прочитай RULES.md.

## Назначение

Скилл готовит полные исходные данные для дальнейшей консультации.

## Стиль формулировок

1. Записывать данные и пояснения простым живым языком, без канцелярита.
2. Если используется медицинская аббревиатура, при первом упоминании давать расшифровку в контексте.

## Входные данные

1. Ответы клиента по анкете
2. references/13_client_work.md
3. references/03_lab_diagnostics.md
4. references/11_reference_values.md

## Последовательность

1. Собрать анкету и анамнез.
2. Сформировать профиль клиента.
3. Составить индивидуальный чек-лист анализов.
4. Дать инструкцию по подготовке к сдаче и по дневнику питания.
5. Подготовить структуру папки клиента.

## Обязательная проверка для /consult

Перед запуском полного анализа агент обязан проверить в clients/[ФИО]/profile.md:
1. Текущие лекарства
2. Жалобы
3. Хронические заболевания и диагнозы
4. Менструальный цикл (для женщин)
5. Курение и алкоголь

Если жалобы и лекарства не указаны:
1. Сначала вывести запрос недостающих данных
2. Не запускать анализ до получения ответа

Исключения:
1. Пользователь явно дал команду "работай с тем что есть"
2. Данные уже переданы inline в текущем сообщении команды

## Выход

Файл clients/[ФИО]/profile.md.
