# Скилл: Сравнение динамики анализов

Перед выполнением — прочитай RULES.md.

## Назначение

Скилл анализирует прогресс клиента между контрольными точками.

## Стиль формулировок

1. Объяснять динамику живым клиническим языком, а не сухим перечислением маркеров.
2. Каждый вывод сопровождать смыслом для клиента: что изменилось и почему это важно.
3. Аббревиатуры расшифровывать в контексте первого упоминания.

## Входные данные

1. Исторические анализы клиента
2. Текущие анализы клиента
3. Последняя схема рекомендаций
4. references/11_reference_values.md

## Последовательность

0. При повторном визите агент СНАЧАЛА читает ВСЕ предыдущие файлы клиента, ПОТОМ сравнивает, ПОТОМ корректирует схему. Никогда не начинает с нуля.
1. Сопоставить одинаковые показатели по датам.
2. Для каждого показателя описать динамику: было, стало, тренд, клинический смысл.
3. Проверить, какие назначения сработали, а какие требуют коррекции.
4. Выявить новые отклонения.
5. Обновить план коррекции и сроки следующего контроля.
6. Записать выводы в learning/cases.md.
