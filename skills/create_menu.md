# Скилл: Составление индивидуального меню

Иерархия источников правды: см. `v2/HIERARCHY.md`.
Перед выполнением — прочитай RULES.md.

## Назначение

Скилл собирает реалистичный рацион с учётом анализов, ограничений и цели.

## Стиль формулировок

1. Объяснять выбор меню живым языком: что это даёт клиенту в его клиническом контексте.
2. Избегать сухих списков без пояснения причин выбора продуктов и режима.
3. Аббревиатуры расшифровывать при первом упоминании.

## Входные данные

1. clients/[ФИО]/profile.md
2. Последняя экспертная рекомендация клиента
3. references/12_menus.md
4. references/02_nutrition_basics.md
5. Профильные references по диагнозам

## Последовательность

1. Определить ограничения по здоровью и пищевой переносимости.
2. Рассчитать калорийность и КБЖУ под цель клиента.
3. Определить структуру приёмов пищи и режим воды.
4. Сформировать меню на N дней с рецептами и порциями.
5. Добавить список покупок и план корректировки на повторном контакте.

## Выход

Файл clients/[ФИО]/[ДАТА]_menu_[N]days.md.
