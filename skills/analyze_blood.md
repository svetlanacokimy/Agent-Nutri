# Скилл: Анализ крови и лабораторных данных

Перед выполнением — прочитай RULES.md.

## Назначение

Скилл выполняет экспертный разбор анализов и формирует аналитическую основу для рекомендаций.

Выполнение строго по этапам ниже: от проверки единиц до системного вывода.

## Входные данные

1. clients/[ФИО]/blood_tests.md и/или фото анализов
2. clients/[ФИО]/profile.md
3. references/11_reference_values.md
4. Профильные references по выявленным отклонениям

## Последовательность

### ПРАВИЛО ОБЯЗАТЕЛЬНЫХ ДАННЫХ

Перед началом полного анализа проверить наличие в clients/[ФИО]/profile.md или clients/[ФИО]/blood_tests.md:
1. Пол
2. Возраст
3. Рост
4. Вес
5. Жалобы
6. Физическая активность

Если отсутствует любой параметр, полный анализ не запускать и сначала выводить запрос:

"Для полноценной консультации необходимы следующие данные, которых нет в файлах клиента:
- Рост: не указан
- Физическая активность: не указана
Пожалуйста, предоставьте эти данные, после чего я выполню полный анализ."

Шаблон запроса нужно адаптировать под фактически отсутствующие поля.

Исключение:
1. Если пользователь явно говорит "работай с тем что есть", выполнить анализ с пометкой об ограничениях.
2. В этом режиме КБЖУ допускается только как приблизительный расчёт с явной пометкой.

### ЭТАП 0 — ПРОВЕРКА ЕДИНИЦ ИЗМЕРЕНИЯ

1. Вызвать skills/unit_conversion.md.
2. Привести все показатели к единицам из references/11_reference_values.md.
3. Не переходить к интерпретации, пока не сняты конфликты единиц.

### ЭТАП 1 — ПОТОЧЕЧНАЯ ОЦЕНКА КАЖДОГО ПОКАЗАТЕЛЯ

Алгоритм чтения каждой строки:
1. Берёшь ячейку "Показатель" — определяешь параметр.
2. Смотришь "Результат" — числовое значение клиента. Если пусто — показатель не сдан, в список "досдать".
3. Смотришь "Референс" — лабораторные границы.
4. Смотришь "Оптимальные значения" из references/11_reference_values.md.
5. Присваиваешь статус:
  - ПАТОЛОГИЧЕСКОЕ ОТКЛОНЕНИЕ — результат вне референса.
  - СУБОПТИМАЛЬНОЕ ЗНАЧЕНИЕ — в референсе, но вне оптимума.
  - В ПРЕДЕЛАХ ОПТИМУМА — всё хорошо.
  - НЕ СДАН — результат отсутствует.
6. Сопоставляешь с симптомами и другими показателями.

Жёсткое правило этапа:
1. НИ ОДИН ПОКАЗАТЕЛЬ НЕ ПРОПУСКАЕТСЯ, даже если "в норме".

### ЭТАП 2 — КЛИНИЧЕСКИЕ СВЯЗКИ

Агент обязан проверить все связки:

1. Анемия:
  - гемоглобин + ферритин + железо + В12 + В9 + трансферрин + MCV + MCH + MCHC + RDW.
2. Щитовидка:
  - ТТГ + Т3св + Т4св + АТ-ТПО + АТ-ТГ + йод + селен + цинк + железо + магний.
3. Инсулинорезистентность:
  - инсулин + глюкоза + HOMA-IR + HbA1c + С-пептид + триглицериды + мочевая кислота.
4. Печень/желчь:
  - билирубин (общ/прям/непрям) + АЛТ + АСТ + коэф. де Ритиса + ГГТ + ЩФ + альбумин.
5. Воспаление:
  - СРБ + лейкоциты + нейтрофилы + СОЭ + лимфоциты + ферритин (как белок острой фазы).
6. Паразитоз:
  - эозинофилы >5% + лимфоциты >37% + нейтрофилы <48% + антитела к паразитам.
7. Белковый обмен:
  - общий белок + альбумин + мочевина + креатинин.
8. Липидный профиль:
  - ОХС + ЛПВП + ЛПНП + триглицериды + КА + гомоцистеин.
9. Копрограмма (если есть):
  - оценить переваривание белков/жиров/углеводов, наличие слизи, крови, лейкоцитов, мышечных волокон, крахмала, жирных кислот.
10. ОАМ (общий анализ мочи, если есть):
  - оценить белок, лейкоциты, эритроциты, бактерии, соли.
11. Ось кишечник-печень:
  - СИБР + инсулинорезистентность + неалкогольная жировая болезнь печени + повышенный ферритин как единый клинический контур.
12. Гормональный контур для женщин:
  - эстрадиол + прогестерон + ДГЭА + кортизол + ГСПГ + пролактин.

### ЭТАП 3 — АВТО-АЛЕРТЫ

Блокирующие (запрещают действия):
1. Ферритин < 20 -> АПК запрещён.
2. Запоры -> АПК запрещён.
3. Беременность/кормление -> АПК запрещён.

Критические (требуют немедленного внимания):
1. Ферритин < 15 -> тяжёлая железодефицитная анемия, срочная коррекция.
2. Инсулин > 25 + глюкоза > 5.5 -> выраженная инсулинорезистентность.
3. ТТГ > 4 или < 0.4 -> направление к эндокринологу.
4. СРБ > 5 -> активное воспаление, искать причину.
5. Гомоцистеин > 10 -> проверить В12, В9, В6.
6. Триглицериды > 5.6 -> риск панкреатита.
7. Ферритин > 300 (мужчины) или > 200 (женщины) -> исключить гемохроматоз, неалкогольную жировую болезнь печени.

Предупредительные:
1. АТ-ТПО повышены + планируется йод -> йод > 150 мкг опасен.
2. Эозинофилы > 5% + лимфоциты > 37% -> вероятен паразитоз.
3. MCV > 95 -> проверить В12, В9.
4. MCV < 80 -> проверить железо, ферритин.
5. Мочевая кислота > 360 -> риск подагры, проверить питание.

### ЭТАП 4 — REASONING CHAINS

1. Для каждого выявленного отклонения строить причинно-следственную цепочку минимум 5-7 звеньев.
2. Искать порочные круги: когда A усугубляет B, а B усугубляет A.
3. Искать корневые причины: одну-две главных, из которых каскадом вытекают остальные нарушения.
4. Описывать прозой, живым академическим языком, с прямой привязкой к показателям клиента.

### ЭТАП 5 — МЕТА-АНАЛИЗ

1. Определить, какие системы организма затронуты, и ранжировать по тяжести.
2. Выделить 1-2 корневые причины каскада.
3. Дать прогноз без коррекции.
4. Дать прогноз при соблюдении рекомендаций через 1/3/6 месяцев.

### ЭТАП 6 — ДОСДАТЬ

1. Сформировать список анализов, которые не сданы, но необходимы.
2. Для каждого анализа дать краткое обоснование "зачем".

### ЭТАП 7 — СИСТЕМНЫЙ ВЫВОД

1. Связать все отклонения в единую клиническую картину.
2. Оформить прозаическим текстом объёмом 20-30 строк.

## Авто-алерты

1. Проверять блокирующие риски до назначения нутрицевтиков.
2. При критических комбинациях выносить алерт в начало разбора.
3. При неполных данных не останавливать работу: формировать вывод на основании имеющихся данных.
4. Блокирующие алерты имеют приоритет над всеми последующими назначениями.
