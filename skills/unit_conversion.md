# Unit Conversion — Скилл пересчёта единиц измерения

## Глобальные правила вывода (ОБЯЗАТЕЛЬНО)

1. Финальные документы (analysis_expert, recommendations_expert, client_report и т.д.) оформляются БЕЗ markdown-таблиц. Формат только структурированный текст.
2. В режиме EXPERT использовать живой академический стиль: объяснять клинический смысл пересчёта и влияние на интерпретацию показателей.
3. При первом упоминании любой аббревиатуры давать полную расшифровку в контексте предложения. Перед выводом — обязательная самопроверка расшифровок.

Примечание: внутренние служебные таблицы допустимы только как технический этап валидации; в итоговый документ они НЕ попадают.

## Когда вызывается

**АВТОМАТИЧЕСКИ** перед каждым запуском `analyze_blood.md`. Без этого скилла анализ **ЗАПРЕЩЁН**.

Порядок вызова:
```
AGENT.md → unit_conversion.md (Этап 0) → analyze_blood.md (Этапы 1–8)
```

## Алгоритм

### Шаг 1. Извлечение единиц

Для **КАЖДОГО** показателя из анализов клиента определить единицу измерения.
Источники единиц: бланк лаборатории (фото), `blood_tests.md`, PDF анализа.

### Шаг 2. Сверка с эталоном

Сравнить единицу клиента с единицей в `references/11_reference_values.md`.
- Если совпадают — пометить ✅, перейти к следующему показателю.
- Если **НЕ совпадают** — перейти к Шагу 3.

### Шаг 3. Пересчёт

1. Найти коэффициент в `references/unit_conversions.md`
2. Пересчитать: `значение_клиента × коэффициент = значение_в_наших_единицах`
3. Записать в таблицу: **исходное (единица клиента) → пересчитанное (наша единица)**
4. Пометить ⚠️

### Шаг 4. Валидация

После пересчёта проверить — попадает ли значение в **физиологически возможный диапазон**.

Контрольные диапазоны (выход за эти пределы = вероятная ошибка):
| Показатель | Невозможно ниже | Невозможно выше |
|---|---|---|
| Гемоглобин (г/л) | 30 | 250 |
| Глюкоза (ммоль/л) | 1,0 | 40 |
| Инсулин (мкЕд/мл) | 0,1 | 500 |
| Витамин D (нг/мл) | 1 | 200 |
| Ферритин (нг/мл) | 0,5 | 10 000 |
| В12 (пг/мл) | 50 | 2 000 |
| ТТГ (мЕд/л) | 0,001 | 100 |
| Холестерин (ммоль/л) | 1,0 | 20 |

Если значение выходит за физиологический диапазон — пометить 🚫 и уведомить пользователя:
> 🚫 **ОШИБКА ПЕРЕСЧЁТА или ОШИБКА ЛАБОРАТОРИИ**: [Показатель] = [значение] [единица] — физиологически невозможное значение. Проверить исходные данные.

### Шаг 5. Формирование итоговой таблицы

Итоговая таблица для передачи в `analyze_blood.md`:

```
| Показатель | Значение клиента | Единица клиента | Наша единица | Пересчитано | Статус |
|---|---|---|---|---|---|
| Инсулин | 61,3 | МЕ/л | мкЕд/мл | 61,3 | ✅ (1:1) |
| В12 | 258 | пмоль/л | пг/мл | 349,6 | ⚠️ пересчёт |
| Т3 св. | 3,34 | пг/мл | пмоль/л | 5,13 | ⚠️ пересчёт |
| MCHC | 340 | г/л | г/дл | 34,0 | ⚠️ пересчёт |
| Гемоглобин | 143 | г/л | г/л | 143 | ✅ |
```

Правила формирования:
- **ВСЕ** показатели включаются в таблицу — не только пересчитанные
- Столбец «Пересчитано» содержит финальное значение **в наших единицах**
- Статус: ✅ (совпало или эквивалент) / ⚠️ (пересчитано) / 🚫 (ошибка)
- Таблица передаётся **целиком** в `analyze_blood.md` для дальнейшей оценки

## Правила (НЕРУШИМЫЕ)

1. **ЗАПРЕЩЕНО** передавать данные в `analyze_blood.md` без прохождения `unit_conversion`
2. **ЗАПРЕЩЕНО** сравнивать числа в разных единицах измерения
3. Если единица **не найдена** в `references/unit_conversions.md` — уведомить пользователя и запросить уточнение. **НЕ** угадывать коэффициент
4. Если значение после пересчёта выходит за физиологический диапазон (Шаг 4) — **ОСТАНОВИТЬ** анализ этого показателя и запросить проверку
5. Таблица пересчёта **ВСЕГДА** включается в начало файла `_analysis_expert.md` — чтобы проверяющий мог верифицировать каждый пересчёт
