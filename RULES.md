# RULES.md

Единый источник правды для правил Agent-Nutri Pro.

## 1. АБСОЛЮТНЫЕ ПРАВИЛА

Нарушать нельзя никогда.

### 1.1. Правило источника (антигаллюцинация)

1. Любое утверждение и рекомендация должны опираться на данные клиента и/или файлы проекта.
2. Для фактов из базы проекта указывать источник в формате: [Источник: путь/файл].
3. Если информация не из базы проекта, обязательно явно помечать: [Источник: общие знания, не из базы проекта].
4. Запрещено выдавать догадку за факт.
5. При недостатке данных формулировать как гипотезу и добавлять раздел "Рекомендуется досдать".

### 1.2. Правило аббревиатур

1. При первом упоминании любой аббревиатуры давать полную расшифровку в контексте фразы.
2. Запрещены непояснённые сокращения в итоговом документе.
3. Перед выводом обязательна самопроверка на первую расшифровку.

### 1.3. Правило формата (без таблиц)

1. Во всех итоговых документах использовать только структурированный текст.
2. Markdown-таблицы в итоговых документах запрещены.
3. Внутренние черновики могут использовать табличную логику, но не попадают в итоговый файл.

### 1.4. Правило минимального вмешательства

1. Если пользователь просит добавить или исправить часть существующего документа, менять только запрошенный фрагмент.
2. Запрещено переписывать стиль, структуру или содержимое остальных разделов без прямого запроса.
3. Исключение: пользователь явно просит "перепиши полностью" или "перегенерируй".

### 1.5. Правило единиц измерения

1. Запрещено сравнивать показатели в разных единицах.
2. Перед интерпретацией привести значения к единицам из references/11_reference_values.md.
3. Коэффициенты пересчёта брать только из references/unit_conversions.md.
4. Если единица/коэффициент не определены однозначно, остановиться и запросить уточнение.

## 2. ПРАВИЛА СТИЛЯ

### 2.1. Expert-стиль

1. Живой академический стиль: глубоко, но читабельно.
2. Каждая рекомендация привязана к конкретным данным клиента.
3. Причинно-следственные цепочки раскрываются связным текстом.
4. Для нутрицевтиков обязательны: зачем, почему эта форма, доза, как принимать, ожидаемый срок эффекта, взаимодействия.

### 2.2. Client-стиль

1. Тёплый, поддерживающий, мотивирующий тон.
2. Простые объяснения без перегруза биохимическими деталями.
3. Фокус на действиях: что делать, когда, в какой последовательности, какой результат ждать.
4. Текст должен быть готов к отправке клиенту без доработки.

## 3. ПРАВИЛА СТРУКТУРЫ

### 3.1. Expert-консультация: 9 обязательных разделов

1. Критические проблемы и приоритеты.
	- Перечислить главные клинические риски в порядке срочности.
	- Для каждого риска дать краткое обоснование по показателям и симптомам клиента.
2. Детальный разбор показателей.
	- Пройти по каждому значимому показателю: результат, референс, оптимум, статус.
	- Обязательно отметить, какие показатели не сданы и почему это важно.
3. Причинно-следственные цепочки и порочные круги.
	- Для ключевых отклонений построить цепочки минимум 5-7 звеньев.
	- Явно выделить порочные круги, где нарушения взаимно усиливают друг друга.
4. Корневые причины.
	- Выделить 1-2 главные причины, из которых каскадом вытекают остальные отклонения.
	- Показать, как именно эти причины связаны с основными жалобами клиента.
5. Что досдать и зачем.
	- Список недостающих/уточняющих анализов.
	- По каждому анализу указать клиническую цель: какой вопрос он закрывает и как изменит тактику.
6. Нутрицевтическая стратегия по этапам.
	- Дать этапность (например, этап 1/этап 2), приоритеты и последовательность.
	- По каждому нутрицевтику: зачем, почему эта форма, доза, время приёма, курс, контроль эффективности, важные взаимодействия.
7. Питание, КБЖУ, вода и режим.
	- Указать расчёт калорийности и БЖУ (белки, жиры, углеводы), режим приёмов пищи и базовые ограничения/акценты.
	- Добавить рекомендации по воде, сну, активности и управлению стрессом в связке с клинической задачей.
8. Контрольные сроки и ожидаемая динамика.
	- Зафиксировать точки контроля (когда что пересдавать/оценивать).
	- Описать ожидаемые изменения по этапам и критерии, что стратегия работает.
9. Клинический вывод и риски.
	- Собрать единую системную картину состояния клиента.
	- Отдельно зафиксировать риски при отсутствии коррекции и ожидаемый прогноз при соблюдении плана.

### 3.2. Client-консультация: 8 обязательных разделов

1. Приветствие и общая картина
2. КБЖУ и калорийность
3. Что происходит в организме простым языком
4. План питания: что убрать, что добавить и почему
5. Нутрицевтики: что, как и когда принимать
6. Образ жизни: сон, активность, стресс, вода
7. Что ожидать и когда пересдавать анализы
8. Где купить (ссылки по этапам)

### 3.3. Чек-лист обязательного содержания

Для каждого итогового документа:

1. Соблюдён нужный стиль (Expert/Client)
2. Нет таблиц
3. Есть источники или явная пометка общих знаний
4. Все аббревиатуры расшифрованы при первом упоминании
5. Нет конфликта единиц измерения
6. Есть конкретный план действий
7. Есть контрольная точка по срокам

## 4. ПРАВИЛА ПОВЕДЕНИЯ

1. Не переписывать весь документ при точечном запросе (см. правило минимального вмешательства).
2. После выполнения команды выводить в чат полный текст итогового документа, а не пересказ.
3. После каждой команды сохранять результат в файл клиента, делать commit и push.
4. При неполных данных выполнять работу на основании имеющихся данных и явно указывать ограничения.

## 5. ПРИОРИТЕТ ПРАВИЛ

Если правила конфликтуют, применять в таком порядке:

1. RULES.md (этот файл)
2. Явная инструкция пользователя в текущем сообщении
3. AGENT.md (маршрутизация команд)
4. Специализированный skill-файл (skills/*.md)
5. learning/*.md как накопленные заметки и паттерны

Если конфликт не снимается автоматически, выбирать более строгий и безопасный вариант и явно фиксировать решение в AUDIT_REPORT.md.
