# v2/ — Ядро второго поколения Agent-Nutri

Эта папка содержит архитектурный слой v2: личность агента, правила стиля, иерархию источников и протоколы поведения. Всё, что написано здесь, главнее `RULES.md` и `skills/*.md`.

---

## Что здесь лежит

**`IDENTITY.md`** — кто такой агент: роль, клиническое мышление, границы компетенции, два режима выхода (EXPERT / CLIENT).

**`STYLE_GUIDE.md`** — единственный источник правды по стилю. Откалиброван по эталонам `1expert_style_example.md.txt` и `1client_style_example.md.txt`. Содержит обязательные слоты для нутрицевтиков, структуры документов, правила языка.

**`HIERARCHY.md`** — иерархия источников правды. Читать первым при любом конфликте правил.

**`KNOWLEDGE_PROTOCOL.md`** — антигаллюцинационный протокол. Три категории знаний, правила цитирования, флаги неопределённости.

**`SOURCES_INDEX.md`** — карта «тема → файл». Где искать знания по конкретной клинической теме в `references/` и `text_extracted/`.

**`PIPELINES.md`** — пайплайны команд. Внутренний процесс мышления агента для каждой команды (`/consult`, `/client`, `/menu`, `/compare`, `/intake`, `/correct`, `/learn`).

**`SELF_LEARNING.md`** — протокол самообучения. Триггеры, куда пишется, формат записи, как применяется при следующих консультациях.

---

## В каком порядке читать

1. `HIERARCHY.md` — чтобы понять, чьё слово главнее
2. `IDENTITY.md` — кто я и как думаю
3. `STYLE_GUIDE.md` — как пишу
4. `KNOWLEDGE_PROTOCOL.md` — откуда беру знания и как помечаю их
5. `PIPELINES.md` — что делаю при каждой команде
6. `SELF_LEARNING.md` — как учусь и исправляю ошибки
7. `SOURCES_INDEX.md` — справочник, обращаться по необходимости

---

## Принцип «эталоны главнее правил»

Два файла в корне репозитория — `1expert_style_example.md.txt` и `1client_style_example.md.txt` — это эталоны реальных хороших ответов агента. Они главнее любых инструкций в `v2/`, `RULES.md` и `skills/*.md`. Если инструкция противоречит эталону — побеждает эталон.

---

## Связь с остальными частями репозитория

`RULES.md` — общие правила. Действуют там, где `v2/` молчит. При конфликте с `v2/` — побеждает `v2/`.

`AGENT.md` — маршрутизатор команд. Использует пайплайны из `v2/PIPELINES.md`.

`skills/*.md` — специализированная логика отдельных шагов. Каждый skill ссылается на `v2/HIERARCHY.md`.

`learning/*.md` — накопленная память. Записывается по правилам из `v2/SELF_LEARNING.md`.

`references/` и `text_extracted/` — база знаний. Структурирована в `v2/SOURCES_INDEX.md`.
