# KNOWLEDGE_PROTOCOL.md — Антигаллюцинационный протокол

Правила работы со знаниями: откуда берём, как помечаем, что делать, когда данных нет.

---

## Три категории знаний

### Категория A — Знания из базы репозитория

Источники: `references/*.md`, `text_extracted/*.txt`, эталоны (`1expert_style_example.md.txt`, `1client_style_example.md.txt`), файлы клиентов.

Правило: после смыслового блока ставить ссылку в формате `[Источник: references/09_nutraceuticals.md]`.

**Важно:** ссылка ставится после смыслового блока (абзаца или группы связанных утверждений), а НЕ после каждого предложения — это убивает читабельность.

Применимость по режимам:
- EXPERT: источники видимы, ставятся явно.
- CLIENT: источники скрыты. Агент внутри себя должен быть уверен, что знание из базы — и только тогда писать его без пометки.

### Категория B — Общее медицинское / биохимическое знание

Определение: факты учебникового уровня, не зависящие от конкретной базы репозитория. Примеры: что такое инсулин, как работает гепсидин, формула Миффлина-Сен-Жеор, референсные нормы из общепринятых источников (ВОЗ, Российские клинические рекомендации), механизмы всасывания витаминов, классическая биохимия цикла Кребса.

Правило: источник не указывается. Агент может добавить пометку `[общее знание]` если сам сомневается в границе между A и B.

Применимость: одинаково в EXPERT и CLIENT.

### Категория C — Знание не из базы и не общее

Определение: свежие мета-анализы с конкретными цифрами, экзотические протоколы, конкретные авторы и исследования, которых нет в `references/`.

Правило: агент **обязан** поставить флаг после такого блока:

```
[⚠ требует верификации: знание не из базы репо — проверьте или удалите этот блок]
```

Затем предложить пользователю: «Хотите оставить это утверждение? Если да — я добавлю источник в `learning/new_knowledge.md`. Если нет — удалю блок.»

Агент **не выдаёт** Категорию C за Категорию A или B без явной пометки.

---

## Что делать, когда знания нет

Если в базе репозитория нет данных по вопросу клиента или по клинической ситуации — агент говорит прямо:

> «Данных по этой теме в базе недостаточно. Для точного ответа нужно досдать / добавить источник / уточнить у специалиста. [⚠ данных в базе нет]»

Агент **не генерирует** правдоподобный ответ вместо признания пробела. «Правдоподобно» не значит «правильно».

---

## Правило конкретных цифр

Любая цифра в документе (доза, концентрация, процент эффективности, срок динамики) должна быть привязана к одному из трёх:

1. **Данным пациента** — «при ферритине 7,98 у этой пациентки»
2. **Источнику из базы** — `[Источник: references/09_nutraceuticals.md]`
3. **Явному ориентиру** — «ориентировочно, без привязки к конкретному источнику»

Если ни одного из трёх — цифру не писать.

---

## Применимость к EXPERT и CLIENT

**EXPERT-режим:**
- Источники видимы после каждого смыслового блока.
- Флаги Категории C обязательны.
- Агент показывает, откуда знает каждое утверждение.

**CLIENT-режим:**
- Источники скрыты (правило из `v2/STYLE_GUIDE.md` и `RULES.md §3.2.1`).
- Но агент должен знать категорию каждого знания — прежде чем писать.
- Знание Категории C в CLIENT-документ не попадает без верификации.

---

## Порядок поиска знаний

При работе над консультацией агент ищет знания в таком порядке:

1. `references/` — выжимки (приоритет при цитировании).
2. `text_extracted/` — первичный материал, если в `references/` нет ответа.
3. Категория B (общее знание) — если тема учебниковая.
4. Категория C — только с флагом верификации.

Карта «тема → файл» — в `v2/SOURCES_INDEX.md`.
