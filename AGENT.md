# Agent-Nutri Pro — Главный маршрутизатор

Перед любой командой сначала прочитай RULES.md.

RULES.md — единый источник правил.
AGENT.md отвечает за маршрутизацию команд и последовательность выполнения.
skills/*.md содержат только специализированную логику конкретного шага.

## Роль

Агент выполняет клинический нутрициологический разбор по данным клиента и базе проекта.

## Пайплайн по умолчанию

1. Прочитать RULES.md.
2. Прочитать входные данные клиента из clients/[Фамилия]/.
3. При необходимости пересчитать единицы по skills/unit_conversion.md.
4. Выполнить профильный skill.
5. Сохранить результат в файл.
6. Обновить learning/cases.md при консультационных командах.
7. Сделать commit и push.
8. Вывести в чат полный текст итогового документа.

## Команды

### /consult [Фамилия]

Полная экспертная консультация.

Порядок:
1. RULES.md
2. skills/unit_conversion.md
3. skills/analyze_blood.md
4. skills/recommend.md

Выходные файлы:
- clients/[Фамилия]/[ДАТА]_analysis_expert.md
- clients/[Фамилия]/[ДАТА]_recommendations_expert.md

### /client [Фамилия]

Клиентская версия на основе последней expert-консультации.

Порядок:
1. RULES.md
2. Последний clients/[Фамилия]/*_recommendations_expert.md
3. skills/recommend.md (адаптация в Client-режим)

Выходной файл:
- clients/[Фамилия]/[ДАТА]_client_report.md

Важно:
- Структура строго по RULES.md: 8 обязательных разделов (включая КБЖУ и ГДЕ КУПИТЬ).
- Если запрос точечный, действует правило минимального вмешательства.

### /compare [Фамилия]

Сравнение динамики между датами.

Порядок:
1. RULES.md
2. skills/compare_dynamics.md

Выходной файл:
- clients/[Фамилия]/[ДАТА]_compare.md

### /menu [Фамилия] [N]

Индивидуальное меню на N дней.

Порядок:
1. RULES.md
2. skills/create_menu.md

Выходной файл:
- clients/[Фамилия]/[ДАТА]_menu_[N]days.md

### /intake [Фамилия]

Первичный приём и профиль клиента.

Порядок:
1. RULES.md
2. skills/client_intake.md

Выходной файл:
- clients/[Фамилия]/profile.md

### /links [Фамилия]

Ссылки на покупку только по назначенным нутрицевтикам.

Порядок:
1. RULES.md
2. Последний clients/[Фамилия]/*_recommendations_expert.md
3. text_extracted/Ссылки БАД.txt

Выход:
- Полный список в чат, структурированным текстом, без таблиц.

### /correct [текст]

Записать коррекцию в learning/corrections.md с датой.

### /learn [текст]

Записать новые знания в learning/new_knowledge.md с датой.

## Правило приоритетов

При конфликте инструкций приоритеты определяются только по RULES.md.
